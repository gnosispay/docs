---
title: "Get paginated transactions for activated cards"
api: "GET /api/v1/cards/transactions"
description: "Retrieves paginated transactions for all activated cards of the authenticated user with clear examples of all transaction kinds"
---

Retrieves paginated transactions for all activated cards of the authenticated user. 

Returns three types of transaction events:
- **Payment**: Regular card transactions (purchases, ATM withdrawals). Can appear immediately after authorization with `isPending: true`, then update when cleared.
- **Refund**: Money returned to your account (product returns, cancellations). Only appears after both authorization and clearing are processed.
- **Reversal**: Cancellation of previous transactions due to technical issues or merchant corrections. Can appear immediately after authorization-level reversals.

The final number might differ slightly, as one thread might contain multiple transactions.

## Request

<ParamField query="cardTokens" type="string">
  Comma-separated list of card tokens
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Maximum number of transactions to return (minimum: 10)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of transactions to skip (minimum: 0)
</ParamField>

<ParamField query="before" type="string">
  Filter transactions before this date (ISO 8601 format, e.g., "2023-04-01T00:00:00Z")
</ParamField>

<ParamField query="after" type="string">
  Filter transactions after this date (ISO 8601 format, e.g., "2023-03-01T00:00:00Z")
</ParamField>

<ParamField query="billingCurrency" type="string">
  Filter by billing currency code
</ParamField>

<ParamField query="transactionCurrency" type="string">
  Filter by transaction currency code
</ParamField>

<ParamField query="mcc" type="string">
  Filter by Merchant Category Code (MCC)
</ParamField>

<ParamField query="transactionType" type="string">
  Filter by transaction type code (e.g., "00" for Purchase, "01" for Withdrawal)
</ParamField>

## Response

<ResponseField name="count" type="integer">
  Total number of transactions available
</ResponseField>

<ResponseField name="next" type="string" nullable>
  URL for the next page of results, null if no more pages
</ResponseField>

<ResponseField name="previous" type="string" nullable>
  URL for the previous page of results, null if on first page
</ResponseField>

<ResponseField name="results" type="array">
  Array of transaction events for the current page. Each transaction has a `kind` field that determines its type and available fields.
</ResponseField>

## Transaction Kinds

## Key Fields by Transaction Kind

### All Transaction Types
- `createdAt`: When the transaction occurred
- `clearedAt`: When it was settled (null if pending)
- `isPending`: Whether awaiting settlement
- `billingAmount`/`billingCurrency`: Amount charged to account
- `transactionAmount`/`transactionCurrency`: Original transaction amount
- `transactionType`: ISO 8583 transaction type code
- `merchant`: Information about where transaction occurred
- `cardToken`: Which card was used

### Payment-Specific Fields
- `status`: Approval status (`"Approved"`, `"IncorrectPin"`, `"InsufficientFunds"`, etc.)

### Refund-Specific Fields  
- `refundAmount`: Amount being refunded
- `refundCurrency`: Currency of the refund

### Reversal-Specific Fields
- `reversalAmount`: Amount being reversed
- `reversalCurrency`: Currency of the reversal

## Integration Notes

### isPending Behavior
- **Payments**: `true` when authorized but not settled, `false` when complete
- **Refunds**: Usually `false` since they require both auth + clearing to appear
- **Reversals**: Typically `false` as they process quickly

### When Transactions Appear
- **Payments**: Immediately after authorization
- **Refunds**: Only after both authorization AND clearing (credit voucher)
- **Reversals**: Timing varies - can be immediate or after clearing
