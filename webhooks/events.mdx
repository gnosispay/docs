title: "Webhook Events Reference"
description: "Complete reference for all Gnosis Pay webhook event types and their data schemas."

This reference provides detailed information about all webhook events sent by Gnosis Pay. Each event includes the event type, description, and complete data schema.

## Event Structure

All webhook events follow this structure:

```json
{
  "eventType": "string",
  "data": {
    // Event-specific data
  }
}
```

## User Lifecycle Events

### user.created

Triggered when a new user account is created in the Gnosis Pay system.

**Event Type**: `user.created`

**Data Schema**:

```json
{
  "id": "string",
  "email": "string",
  "phone": "string",
  "firstName": "string",
  "lastName": "string",
  "address1": "string",
  "address2": "string",
  "city": "string",
  "postalCode": "string",
  "state": "string",
  "country": "string",
  "nationalityCountry": "string",
  "signInWallets": [
    {
      "address": "string"
    }
  ],
  "safeWallets": [
    {
      "address": "string",
      "chainId": "string"
```

## KYC Events

### kyc.status.changed

Triggered when a user's KYC verification status changes.

**Event Type**: `kyc.status.changed`

**KYC Status Values**:

- `notStarted` - KYC process not initiated
- `documentsRequested` - Documents requested from user
- `pending` - Documents submitted, under review
- `processing` - Documents being processed
- `approved` - KYC approved
- `resubmissionRequested` - Additional documents needed
- `rejected` - KYC rejected
- `requiresAction` - User action required

**Data Schema**: Same as `user.created`

**Example**:

```json
{
  "eventType": "kyc.status.changed",
  "data": {
    "id": "user_123",
    "email": "john.doe@example.com",
    "phone": "+1234567890",
    "firstName": "John",
    "lastName": "Doe",
    "address1": "123 Main Street",
    "address2": "Apt 4B",
    "city": "New York",
    "postalCode": "10001",
    "state": "NY",
    "country": "US",
    "nationalityCountry": "US",
    "signInWallets": [
      {
        "address": "0x1234567890abcdef1234567890abcdef12345678"
      }
    ],
    "safeWallets": [
      {
        "address": "0xabcdef1234567890abcdef1234567890abcdef12",
        "chainId": "100"
      }
    ],
    "kycStatus": "approved",
    "availableFeatures": {
      "moneriumIban": true
    },
    "cards": [
      {
        "id": "card_123",
        "cardToken": "123456789",
        "lastFourDigits": "1234",
        "embossedName": "JOHN DOE",
        "activatedAt": "2024-01-15T10:30:00Z",
        "virtual": false
      }
    ],
    "bankingDetails": {
      "moneriumIban": "GB29NWBK60161331926819",
      "moneriumBic": "NWBKGB2L",
      "moneriumIbanStatus": "ACTIVE",
      "address": "123 Main Street, New York, NY 10001"
    },
    "isSourceOfFundsAnswered": true,
    "isPhoneValidated": true
  }
}
```

## Banking Events

### iban.created

Triggered when an IBAN is created for a user.

**Event Type**: `iban.created`

**Data Schema**:

```json
{
  "iban": "string",
  "bic": "string"
}
```

**Example**:

```json
{
  "eventType": "iban.created",
  "data": {
    "iban": "GB29NWBK60161331926819",
    "bic": "NWBKGB2L"
  }
}
```

### iban.status.changed

Triggered when an IBAN's status changes.

**Event Type**: `iban.status.changed`

**IBAN Status Values**:

- `NOTSTARTED` - IBAN creation not started
- `PENDING` - IBAN creation pending
- `PENDING_OAUTH` - OAuth authorization pending
- `ASSIGNED` - IBAN assigned and active

**Data Schema**:

```json
{
  "previousStatus": "string",
  "currentStatus": "string"
}
```

**Example**:

```json
{
  "eventType": "iban.status.changed",
  "data": {
    "previousStatus": "PENDING",
    "currentStatus": "ASSIGNED"
  }
}
```
